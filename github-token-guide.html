🔑 GitHub Token 配置指南

解决"创建Gist失败"问题，正确配置GitHub Personal Access Token

🔍 问题诊断

常见错误原因：

  • Token权限不足（缺少gist权限）
  • Token已过期或被撤销
  • Token格式错误
  • 网络连接问题
  • GitHub API限制

解决步骤：

  1. 重新生成GitHub Personal Access Token
  2. 确保选择正确的权限（gist权限）
  3. 复制完整的Token（以ghp_开头）
  4. 在同步管理页面重新配置
  5. 测试Token有效性

📋 获取GitHub Token

1

访问GitHub设置

登录GitHub，点击右上角头像 → Settings

🔗 打开GitHub设置
2

进入Token设置

左侧菜单 → Developer settings → Personal access tokens → Tokens (classic)

🔗 打开Token设置
3

生成新Token

点击"Generate new token (classic)"

4

配置Token

Note: 云工坊同步Token

Expiration: 建议选择"90 days"或更长

权限选择:

  • ✅ gist - 创建和管理Gist（必需）
  • ✅ repo - 访问仓库（可选，用于备份）
5

生成并复制Token

点击"Generate token"，立即复制生成的Token（以ghp_开头）

⚠️ 重要提醒：

Token只显示一次，请立即复制保存！

⚙️ 配置Token

1

访问同步管理页面

打开GitHub同步管理页面

2

输入Token

在"GitHub Personal Access Token"字段中粘贴您的Token

Token格式示例：ghp_xxxxxxxxxxxxxxxxxxxx

3

初始化同步

点击"初始化同步"按钮

系统会自动创建Gist并保存配置

4

保存Gist ID

初始化成功后，请保存显示的Gist ID

Gist ID用于在其他设备上连接同一个数据源

🛠️ 故障排除

错误：401 Unauthorized

原因：Token无效或已过期

解决：重新生成Token，确保选择gist权限

错误：403 Forbidden

原因：Token权限不足

解决：确保Token包含gist权限

错误：422 Unprocessable Entity

原因：Gist创建参数错误

解决：清除浏览器缓存，重新尝试

错误：Network Error

原因：网络连接问题

解决：检查网络连接，稍后重试
